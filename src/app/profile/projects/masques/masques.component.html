VERIFICATION DU PORT DU MASQUE GRACE AU DEEP LEARNING

Fin : Mars 2020

Durée : 2 semaines

Technologies :

Tags :

Python, DeepLearning, Jupyter

Contexte :

Dans le cadre de mon échange Erasmus dans l'université de Luleå en Suède, j'ai participé au cours appelé "Applied Artificial Intelligence". Au cours de ce dernier, j'ai eu l'occasion d'approfondir davantages mes connaissances dans diverses techniques incontournables utilisées dans le domaine du Machine Learning : Réseaux de neurones récurrents , Traitement du language naturel, Réseau de neurones par convolution, ... Mais j'ai également pu découvrir des techniques plus avancées que je n'avais auparavant pas utilisé comme par exemple les SOM (Self-Organized Maps). A la fin de ce cours, l'un des objectifs fut de mettre en place un projet utilisant des concepts vus lors des cours: backpropagation, apprentissage profond, ...

Présentation du projet :

Pour ce projet, j'ai décidé de travailler sur un projet utilisant les CNN (Convolutionnal Neural Networks) car c'est une technique qui m'intéresse beaucoup. J'ai ainsi cherché une base de données sur le site kaggle contenant des images que je puisse utiliser afin de créer un modèle. Je suis finalement tombé la base de données suivante : Detection de masques qui consiste en une bibliothèque d'images contenant des personnes qui soit portent le masque correctement, soit le portent mal, soit ne le portent pas du tout.
Sur chacune de ces images, il y a une ou plusieurs personnes. La base contient également des fichiers au format .XML contenant les coordonnées des découpes de l'image afin de ne garder que les visages des personnes sur l'image globale ainsi que l'étiquettage lié à la personne en question (masque bien porté, masque mal porté ou pas de masque).
Afin d'avoir une vue globale du projet, vous pouvez accéder au Jupyter du projet.

Pré-traitement de la base :

La première étape du projet fut de pré-traiter la base de données en extrayant chaque face puis en attribuant à ces images les bons labels. Après le traitement, je disposais de deux tableaux : un contenant les images, un second contenant les labels associés.
Les images étant de tailles différentes, il a également fallu les redimensionner afin que toutes aient une taille identique. Enfin, afin de classifier les images pour la phase d'entrainement, les labels ont étés encodés en vecteurs one-hot.

Phase d'apprentissage :

L'architecture choisie pour l'apprentissage fut très simple: 3 couches de convolution chacune suivies d'une rectification à l'aide d'une fonction de Leaky ReLU, d'une couche de MaxPoooling et enfin d'une couche de Dropout (afin d'éviter le sur-apprentissage).
Afin d'avoir plus d'informations sur le CNN et aussi davantages de détail sur l'architecture, je vous conseille de jeter un oeil au Jupyter du projet.
Je détaille également dans ce Notebook comment j'ai étudié différentes architectures afin de trouver celle obtenant les meilleurs résultats possibles.

Bilan :

Après avoir trouvé la meilleur architecture possible (suite à une série de tests), j'ai décidé de tenter de classifier des photographies prises de moi même portant le masque ou pas. Le résultat obtenu fut le suivant : le modèle est capable de classifier avec une très grande précision les images de personnes portant un masque ou n'en portant pas (précision > 98%), cependant, la plupart des images contenant quelqu'un portant un masque de la mauvaise manière ne sont pas classifiées dans la bonne catégorie.
Peut être qu'une augmentation des données pourrait permettre d'obtenir de meilleurs résultats (transformation, ajout de données, ...).
